1 The Problem

Satellites in low Earth orbit face atmospheric drag from the thermosphere. During geomagnetic storms, the thermosphere expands unpredictably, increasing drag in ways SGP4 cannot anticipate. Solar wind at L1 arrives ~45 min before impacting Earth — a natural leading indicator.

Question: Can a neural network learn the solar wind to orbit perturbation relationship?

Answer: Yes. LSTM achieves 125 km MAE at 6h (54.5 km at 1h), beating Kepler by 4.6x. Adding solar wind via cross-attention improves storm predictions by 17%.

3 The Models

Bidirectional LSTM (Best Overall)

class OrbitLSTMDirect(nn.Module):
    def __init__(self, input_dim=6, hidden_dim=128, num_layers=3,
                 horizon=360, output_dim=3):
        self.lstm = nn.LSTM(input_dim, hidden_dim, num_layers,
                           batch_first=True, bidirectional=True)
        self.fc = nn.Sequential(
            nn.Linear(hidden_dim * 2, hidden_dim), nn.GELU(),
            nn.Linear(hidden_dim, horizon * output_dim))

    def forward(self, x):
        _, (h, _) = self.lstm(x)
        h = torch.cat([h[-2], h[-1]], dim=-1)
        return self.fc(h).view(-1, 360, 3)

Residual Gated Multi-Modal (Novel Architecture)

Core idea: output = base + gate * perturbation

Architecture

Orbit encoder (BiLSTM) produces base prediction

Solar wind encoder (BiLSTM) + cross-attention produces perturbation

Sigmoid gate controls correction strength per timestep

Guarantee: Gate initialized near zero; model starts as pure LSTM

def forward(self, orbit_input, solar_input):
    # Orbit branch -> base prediction
    orbit_out, (h, _) = self.orbit_enc(self.orbit_proj(orbit_input))
    h_cat = torch.cat([h[-2], h[-1]], dim=-1)
    base = self.base_head(h_cat)

    # Solar wind -> cross-attended perturbation
    s = self.solar_enc(self.solar_proj(solar_input))
    attended = self.cross_attn(orbit_out, s)
    summary = (attended * softmax(self.attn_weight(attended))).sum(dim=1)
    perturbation = self.perturbation_head(summary)
    gate = sigmoid(self.gate_net(h_cat))

    return base + gate * perturbation

Two-Phase Training

# Phase 1 (20 epochs, LR=1e-3): Freeze solar branch
model.freeze_solar_branch()  # Only orbit encoder trains

# Phase 2 (80 epochs, LR=1e-4): Unfreeze all
model.unfreeze_all()  # Solar branch learns corrections
# Without Phase 1, convergence fails above 2,000 km MAE
Why two-phase? Solar wind gradients destabilize the orbit encoder early on. Phase 1 establishes a stable baseline. Phase 2 fine-tunes at 10x lower LR.

6 Negative Results

13-Feature Expansion: More Features = More Noise

Added AL, AU indices + clock angle (sin/cos) + dynamic pressure. F10.7 excluded (daily cadence, leakage risk).

Result: 181 km MAE — worse than 8-feature model (163 km). Original features already capture the relevant signal.

DSCOVR: Solar Wind Hurts at L1

Multi-modal: 25,059 km vs LSTM: 12,797 km. No atmosphere at L1, so drag corrections are pure noise.
